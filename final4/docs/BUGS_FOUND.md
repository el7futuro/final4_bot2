## Ошибка 1: Структура хода

**Текущая реализация:**
- Каждый игрок ходит по очереди (сначала один, потом другой)
- Каждый бросает кубик отдельно

**Правильно:**
- ОБА игрока делают ставки одновременно в рамках одного хода
- ОДИН бросок кубика определяет результат для ОБОИХ

**Статус:** 🟡 Частично исправлено
- ✅ TurnState обновлён для хранения ставок обоих игроков
- ❌ GameEngine.roll_dice() требует доработки

---

## Ошибка 2: Количество ставок за ход

**Текущая реализация:**
- Можно сделать 1-N ставок на разных игроков
- Вторая ставка опциональна

**Правильно:**
- Ход 1 (вратарь): РОВНО 1 ставка
- Ходы 2-11 (полевые): РОВНО 2 ставки на ОДНОГО игрока
- Вторая ставка ОБЯЗАТЕЛЬНА

**Статус:** 🟡 Частично исправлено
- ✅ TurnState.get_required_bets_count() добавлен
- ❌ Валидация обязательности 2 ставок требует доработки

---

## Ошибка 3: Доступные игроки для ставок

**Текущая реализация:**
- Только игроки с `is_on_field=True` (11 игроков)

**Правильно:**
- ВСЕ 16 игроков заявки доступны для ставок
- Не путать "на поле" и "в заявке"

**Статус:** ✅ ИСПРАВЛЕНО
- Match.get_available_players_for_betting() теперь использует team.players (все 16)

---

## Ошибка 4: Карточка Свисток

**Текущая реализация:**
- Карточка вытягивается опционально (`draw_card` вызывается отдельно)

**Правильно:**
- При выигрыше ХОТЯ БЫ ОДНОЙ ставки — карточка вытягивается АВТОМАТИЧЕСКИ
- Это часть процесса броска кубика

**Статус:** ❌ Требует исправления в GameEngine.roll_dice()

---

## Ошибка 5: Начисление действий вратарю

**Было в ручной симуляции:**
- Начислил +1 отбитие вместо +3

**Правильно (по коду ActionCalculator):**
- Вратарь при выигрыше Чёт/Нечёт → +3 отбития

**Статус:** ✅ Код правильный, ошибка была только в текстовой симуляции

---

## Ошибка 6: Две ставки одного типа ЗАПРЕЩЕНЫ

**Текущая реализация:**
- Можно делать 2 ставки любого типа на одного игрока

**Правильно:**
- НЕЛЬЗЯ 2 ставки одного типа (например, Чёт/Нечёт + Чёт/Нечёт)
- МОЖНО комбинации разных типов:
  - Чёт/Нечёт + Больше/Меньше ✅
  - Чёт/Нечёт + Точное число ✅
  - Больше/Меньше + Точное число ✅
  - Точное число + Точное число — возможно для определённых позиций (уточнить)

**Статус:** ❌ Требует добавления валидации в BetTracker

## Ошибка 7: Не проверяются лимиты перед показом вариантов

**Текущая реализация:**
- Показываю все типы ставок, потом проверяю лимиты

**Правильно:**
- СНАЧАЛА проверить все лимиты (голы защитников, полузащитников, форвардов, чёт/нечёт)
- ПОТОМ показать ТОЛЬКО доступные варианты
- НЕ предлагать то, что недоступно

**Статус:** ❌ Требует исправления в UI/логике

---

## Ошибка 8: КРИТИЧЕСКАЯ — Игрок доступен только если есть 2 разных типа ставок

**Текущая реализация:**
- Показываю всех игроков, у которых is_available=True и не использованы

**Правильно:**
- Игрок доступен ТОЛЬКО если у него есть минимум **2 РАЗНЫХ типа ставок**
- Если из-за лимитов остался только 1 тип — игрок **НЕДОСТУПЕН**
- Проверка должна быть ДО показа списка игроков
- Нужно моделировать игру до конца 11 хода, чтобы не попасть в тупик

**Пример:**
- Форварды: Чёт/Нечёт ❌, Больше/Меньше ✅, Точное число (лимит 4)
- Когда лимит 4 исчерпан → у форварда остаётся 1 тип → форвард НЕДОСТУПЕН

**Статус:** ❌ КРИТИЧЕСКАЯ ОШИБКА — требует исправления
